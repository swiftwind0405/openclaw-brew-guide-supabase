# 从图片新增咖啡豆

用户发送咖啡豆包装/标签图片时，识别图片中的咖啡豆信息，补全缺失字段后调用 `brew_guide_upsert_bean` 工具写入记录。

## 字段定义

### 必填字段

| 字段 | 键名 | 说明 | 示例 |
|------|------|------|------|
| 豆名 | `name` | 咖啡豆名称，通常是产品名 | 花魁 6.0、瑰夏村日晒 |

### 可选字段

| 字段 | 键名 | 说明 | 示例 |
|------|------|------|------|
| 产地 | `origin` | 产国或产区 | 埃塞俄比亚 古吉、巴拿马 波奎特 |
| 烘焙商 | `roaster` | 烘焙商/品牌名 | SEE SAW、M2M |
| 处理法 | `process` | 生豆处理方式 | 日晒、水洗、蜜处理、厌氧日晒 |
| 品种 | `variety` | 咖啡品种 | 瑰夏、铁皮卡、74110、SL28 |
| 烘焙度 | `roastLevel` | 烘焙程度 | 浅烘、中浅烘、中烘、中深烘、深烘 |
| 购买日期 | `purchaseDate` | ISO 日期字符串 | 2026-03-31 |
| 烘焙日期 | `roastDate` | ISO 日期字符串 | 2026-03-25 |
| 备注 | `notes` | 风味描述或其他备注 | 柑橘、茉莉花、蜂蜜、发酵感 |

## 工作流程

### 第一步：识别图片信息

仔细观察图片，提取所有可识别的咖啡豆信息。常见的信息来源：

- **包装正面**：品牌名、产品名、产地
- **包装背面/标签**：品种、处理法、烘焙度、烘焙日期、风味描述
- **价签/吊牌**：价格、克重、烘焙日期
- **产品卡片**：完整信息集合

识别完成后，将结果整理为结构化列表呈现给用户。

### 第二步：校验并补全信息

将识别结果以清晰的列表展示给用户，格式如下：

```
📋 从图片中识别到的咖啡豆信息：

✅ 豆名：花魁 6.0
✅ 产地：埃塞俄比亚 古吉
✅ 烘焙商：XXX Coffee
✅ 处理法：日晒
✅ 品种：74110
✅ 烘焙度：浅烘
✅ 烘焙日期：2026-03-20
⬜ 购买日期：未识别
⬜ 备注：未识别

请确认以上信息是否正确？如需修改或补充，请告诉我。
```

**关键规则：**

- `name`（豆名）为必填字段。如果图片中无法识别出豆名，**必须**向用户询问。
- 所有可选字段如果无法识别，标记为"未识别"即可，用户可以选择不补充。
- 不要凭空编造任何字段值。无法确定的信息就标记为未识别。
- 如果图片中有多种语言（如中英文），优先使用中文值，但保留英文作为参考。

### 第三步：确认并写入

用户确认信息无误后（或补充完必填字段后），调用 `brew_guide_upsert_bean` 工具写入。

调用示例：

```json
{
  "bean": {
    "name": "花魁 6.0",
    "origin": "埃塞俄比亚 古吉",
    "roaster": "XXX Coffee",
    "process": "日晒",
    "variety": "74110",
    "roastLevel": "浅烘",
    "roastDate": "2026-03-20"
  }
}
```

- 只传有值的字段，不传未识别的字段。
- `id` 不需要传，插件会自动生成。
- 写入成功后，告知用户操作结果。

### 异常处理

- **图片不清晰**：告知用户图片模糊，请求发送更清晰的图片或手动补充信息。
- **不是咖啡豆图片**：礼貌告知用户图片中未识别到咖啡豆信息，询问是否手动录入。
- **写入失败**：将错误信息反馈给用户，建议重试或检查网络连接。
